## 一、简历撰写核心原则

### 1.1 STAR 量化原则

每段项目经历都要遵循 STAR 法则并尽量量化成果：

- **S（Situation）**：项目背景——什么场景、什么规模
- **T（Task）**：你的任务——负责什么、目标是什么
- **A（Action）**：你的行动——用了什么技术、做了什么决策
- **R（Result）**：成果——提升了多少、解决了什么问题

**反面示例**（空洞无力）：
> 负责项目开发工作，参与了相关功能的设计和实现。

**正面示例**（具体有力）：
> 主导设计并实现了基于 ReAct 循环的 AI Agent 系统，集成 12 个 MCP 工具，支撑日均 5000+ 次对话请求，用户问题解决率从 60% 提升至 85%。

### 1.2 六大黄金法则

1. **数字说话**：尽量用数字量化每一条成果（提升 XX%、减少 XX 毫秒、覆盖 XX 个场景）
2. **技术栈明确**：在每个项目中明确列出使用的技术栈
3. **深度优于广度**：2 个有深度的项目 > 5 个浅尝辄止的项目
4. **关键词匹配**：根据目标 JD 调整关键词的顺序和权重
5. **「做了什么」不如「解决了什么问题」**：结果导向，不是任务清单
6. **一页为佳**：除非有 10 年+经验，简历控制在一页 A4 纸内

### 1.3 AI Agent 方向简历的特殊性

AI Agent 方向的简历与传统后端开发简历的不同之处：

- **源码理解加分**：面试官特别看重候选人是否读过 Agent 框架的源码，而非只是调用 API
- **协议理解加分**：对 MCP 等协议的理解比单纯会用某个框架更有含金量
- **端到端经验**：从 Prompt 设计到系统部署的全链条经验比单点能力更有竞争力
- **开源贡献**：有 GitHub 贡献或技术博客会显著提升简历吸引力

## 八、简历注意事项

### 8.1 必须避免的错误

| 错误类型 | 错误示例 | 正确做法 |
|---------|---------|---------|
| 空洞描述 | "负责项目的开发工作" | 用 STAR 法则具体描述 |
| 缺少量化 | "提升了系统性能" | "响应时间从 5s 降低到 2s" |
| 技术堆砌 | 列出 20 个技术但无深度 | 精选 8-10 个有实践经验的 |
| 过度夸大 | "精通 xxx"（实际只用过一次） | 用「了解/熟悉/精通」分级 |
| 格式混乱 | 字体大小不一、对齐混乱 | 统一格式、简洁清爽 |
| 信息遗漏 | 没有联系方式或 GitHub | 确保基本信息完整 |
| 篇幅过长 | 简历超过 2 页 | 应届生 1 页，社招 1-2 页 |
| 敏感信息 | 附上身份证号/家庭住址 | 只提供必要的联系信息 |

### 8.2 项目经历的描述规范

**每个项目应包含**：
1. 项目名称 + 时间段
2. 技术栈列表（具体版本可省略）
3. 3-5 条核心职责描述（用动词开头：设计、实现、优化、研究）
4. 1-2 条量化成果

**动词使用建议**：
- 高级感动词：主导、设计、架构、研究、优化
- 中级动词：实现、开发、集成、搭建、完成
- 避免使用：参与、协助、帮助（显得没有主导权）

### 8.3 技能熟练度分级标准

| 级别 | 含义 | 示例 |
|------|------|------|
| 精通 | 能讲出底层原理，有大量实践经验 | Python（5 年+，读过 CPython 源码） |
| 熟悉 | 有项目经验，能独立完成开发任务 | FastAPI（2 个项目，含中间件开发） |
| 了解 | 学习过原理，有简单使用经验 | Go（完成过一个小项目） |

> ⚠️ 重要提醒：「精通」是一个很重的词，面试官会围绕你标注为「精通」的技术深入追问。如果不确定，宁可写「熟悉」也不要写「精通」。

### 8.4 针对 ATS（简历筛选系统）的优化

大厂通常使用 ATS 自动筛选简历，优化策略：

1. **使用标准格式**：避免花哨的排版、图标、图片，使用纯文本友好的格式
2. **关键词密度**：确保 JD 中的核心关键词在简历中出现 2-3 次（自然融入）
3. **标准标题**：使用常见的章节标题（教育背景、项目经历、技术技能）
4. **PDF 格式**：投递时使用 PDF 格式，避免 Word 格式在不同系统上显示错乱
5. **文件命名**：`姓名-岗位名-联系方式.pdf`（如 `张三-AI Agent开发工程师-138xxxx.pdf`）

## 总结

简历是面试的第一道门槛，核心要做到：

1. **精准匹配**：根据目标岗位调整关键词和项目侧重
2. **量化说话**：用数字证明你的价值，避免空洞描述
3. **深度取胜**：展示源码级理解、协议理解等深度能力
4. **格式专业**：简洁、清晰、一页为佳
5. **持续更新**：随着项目进展和学习深入持续完善简历

> 💡 **最重要的原则**：简历中写的每一条都要做好被面试官深挖的准备。宁可少写两条也不要写自己答不上来的内容。

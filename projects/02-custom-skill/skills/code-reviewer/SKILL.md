# 代码审查技能

## 使用方法
当用户请求代码审查时，按以下步骤执行：

1. 读取用户提供的代码文件
2. 分析代码质量（复杂度、可读性、安全性）
3. 生成审查报告

## 审查维度
- **安全性**: SQL注入、XSS、敏感信息泄露
- **性能**: 时间复杂度、内存使用、N+1查询
- **可维护性**: 命名规范、函数长度、注释质量
- **最佳实践**: 设计模式、错误处理、测试覆盖

## 输出格式
```
### 代码审查报告
- 严重问题: X 个
- 警告: X 个
- 建议: X 个

#### 详细问题列表
1. [严重] 文件:行号 - 问题描述
   建议修复: ...
```
